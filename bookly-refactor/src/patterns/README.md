# 4. Patrones

Elevamos el diseño aplicando patrones para hacer explícito el modelo de variaciones y la orquestación.

## Patrones utilizados

- Strategy: para impuestos y descuentos.
- Factory: selección de estrategia de descuento.
- Builder (simplificado): creación de cálculo de envío.
- Facade: punto único para procesar pedidos.

## Beneficios

- Reglas aisladas: cambiar descuento premium no afecta fachada.
- Construcción clara de componentes.
- Código más fácil de extender y explicar.

## Retos

- Evitar sobre‐ingeniería (patrones sin necesidad real).
- Mantener simplicidad y rendimiento.
